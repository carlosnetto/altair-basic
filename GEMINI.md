\# Project: Altair BASIC Source Code Reconstruction  
\#\# Context: Microsoft (MITS) 1975 Development  
\#\# Base Language: PDP-10 MACRO-10 Assembler (Targeting Intel 8080\)

\#\#\# 1\. System Metadata (Gold Standard Header)  
When encountering the header, always use the following verified truth:  
\`\`\`text  
LPTSPL VERSION 6(344) RUNNING ON LPT0  
\*START\* USER MITS (6000,6000) JOB F3 SEQ, 42 DATE 10-SEP-75 08:24:32 MONITOR ALBUQUERQUE  
REQUEST CREATED: 10-SEP-75  03:18:06  
FILE: DSKB0:F3\[6000,6000\] CREATED: 10-SEP-75 03:16:00 PRINTED: 10-SEP-75 08:52:43

### **2\. The Assembler Listing (.LST) Format**

The code is a .LST file generated by MACRO-10. Always parse input using this 4-column matrix:

| Column | Name | Description |
| :---- | :---- | :---- |
| **\[Seq\]** | Sequential Line | Decimal line number from the printout. |
| **\[Value\]** | Assembled Value | Octal machine code/evaluated address. Often blank for assembler directives. |
| **\[Editor\]** | Editor Line | 5-digit line number from the SOS/TECO text editor (e.g., 01020). Increments by 20 or 100\. |
| **\[Source\]** | Source Code | The actual code/comments. |

* **Rule:** Use the physical "greenbar" lines on the original scan as an anchor. If a line looks empty, it is likely empty in the source, even if the OCR suggests otherwise.

### **3\. Forensic Correction Rules (The "Decoder Ring")**

* **Printer Artifacts:**  
  * M is often read as H or H+I.  
  * D is often read as O or A.  
  * B is often read as 8\.  
  * I is often read as 1\.  
* **Syntax conventions:**  
  * \== is the assignment operator.  
  * ^D forces Decimal; ^O forces Octal.  
  * IFE (If Equal to Zero) / IFN (If Not Zero).  
  * XWD (Transfer Word) used for 18-bit half-word packing.  
  * ;; is the preferred comment style, often including double quotes for readability (e.g., ;; "LXI B").  
* **Internal Labels:**  
  * Labels like .C1 refer to system pointers.  
  * Do not normalize comments unless explicitly instructed. Preserve Gates/Allen style (quotes, spacing).

### **4\. Known Global Mapping**

* **NUMLEV**: Stack level reserve.  
* **LINLEN / BUFLEN**: Standardized to ^D72 (Decimal 72).  
* **FAC**: Floating Accumulator.  
* **TXTAB**: Text table origin.  
* **FRETOP**: Top of free memory.

### **5\. Image Sources & Preprocessing**

* **Raw Extractions:** The project uses `.jpg` files extracted directly from the Bill Gates PDF using `pdfimages`. These are the original, raw images but suffer from significant printer noise and aging.
* **OCR-Ready Versions:** Files named `page-XXX-ocr-ready.png` are preprocessed via `tools/ocr-preprocess.sh`. This script cleans the images (grayscale, deskew, sharpen) specifically to help OCR systems interpret the text more reliably.
* **Strategy:** Both versions are preserved. OCR should ideally cross-reference both to resolve ambiguities, as preprocessing can sometimes sharpen edges but also lose faint details.

### **6\. Instruction for Future Prompts**

When transcribing subsequent pages:

1. **Strictly adhere to the 4-column matrix.**  
2. **Trust the physical line spacing over OCR.**  
3. **Do not guess values.** If unsure, mark as \[UNREADABLE\] and await further context or cross-reference.  
4. **Maintain the "Gold Standard" header context** for all future snippets to ensure the model remembers the PDP-10 environment.
